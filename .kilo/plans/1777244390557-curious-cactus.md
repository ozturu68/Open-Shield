## Kritik Sorunlar (Hemen Düzeltilmeli)

### 1. security.js:64 `debugger` İfadesi

**Şiddet:** CRITICAL
**Dosya:** `src/security.js:64`

```javascript
function detectAntiDebug() {
  debugger;  // ← ÜRETİM KODUNDA BU OLMAMALI!
  ...
}
```

**Etki:** DevTools açıkken uzantı çalışması durur, kullanıcı deneyimi bozulur.

**Çözüm:**
```javascript
function detectAntiDebug() {
  // Debug check without breakpoint
  const devToolsOpen = (window.outerWidth - window.innerWidth > 100) ||
                       (window.outerHeight - window.innerHeight > 100);
  if (devToolsOpen) {
    // Log or handle silently
    return true;
  }
  return false;
}
```

### 2. webrtc.js vs background.js IP Filtreleme Uyuşmazlığı

**Şiddet:** HIGH
**Dosyalar:** `src/webrtc.js`, `src/background.js:119-166`

**Sorun:** İki farklı `RTCPeerConnection` wrap implementasyonu var. `webrtc.js` eksik private IP kontrolü yapıyor:

```javascript
// webrtc.js:26 - EKSİK kontrol
if (ip.startsWith('192.168.') || ip.startsWith('10.') || ip.startsWith('172.')) {
```

```javascript
// background.js:119-166 - TAM kontrol
function isPrivateIPv4(ip) {
  const p = ip.split('.').map(Number);
  return p[0] === 10 || p[0] === 172 && p[1] >= 16 && p[1] <= 31 ||
         p[0] === 192 && p[1] === 168 || p[0] === 127 || p[0] === 0;
}
```

**Çözüm:** `webrtc.js`'i `background.js`'deki tam implementasyonla değiştir veya `webrtc.js`'i tamamen kaldırıp sadece background.js'deki versiyonu kullan.

## Mimari İhlaller (Orta Öncelik)

### 4. options.js Doğrudan DNR API Erişimi

**Şiddet:** MEDIUM
**Dosya:** `options/options.js:88-89, 114-120`

```javascript
// YANLIŞ - doğrudan DNR erişimi
await chrome.declarativeNetRequest.updateEnabledRulesets({
  [cb.checked ? "enableRulesetIds" : "disableRulesetIds"]: [l.id]
});
```

**Sorun:** Mimari kurallara göre UI → Background → DNR zinciri bozuluyor.

**Çözüm:** Mesaj göndererek background.js üzerinden işlem yap:
```javascript
chrome.runtime.sendMessage({
  type: 'SET_RULESET',
  rulesetId: l.id,
  enabled: cb.checked
});
```

## Güvenlik Endişeleri

### 7. Runtime Filter Güncelleme

**Dosya:** `src/background.js:666-675`

Uzantı çalışırken harici GitHub/AdGuard sunucularından ABP filter listesi indiriliyor.

**Risk:** Yabancı kod yüklenmesi, potansiyel supply-chain saldırısı.

**Öneri:** Filtreleri build-time statik olarak derle, runtime güncelleme devre dışı bırak veya imza doğrulama ekle.

---

### 8. browsingData Permission

**Öneri:** Kullanım sıklığını analiz et, gerekmiyorsa kaldır.

## Performans İyileştirmeleri

### 9. MutationObserver Optimizasyonu

**Dosya:** `src/cosmetic.js`

```javascript
// Mevcut: Her mutation için tüm selector'ları kontrol ediyor
observer.callback = (mutations) => {
  for (const m of mutations) {
    for (const sel of documentSelectors) { // O(n*m)
      // ...
    }
  }
};
```

**Öneri:** Debounce veya batched processing ekle.

---

### 10. requestIdleCallback Fallback

**Dosya:** `src/cosmetic.js`

```javascript
// Fallback yok
const ro = new ResizeObserver(() => requestIdleCallback(processQueue));
```

**Öneri:** setTimeout fallback ekle:
```javascript
const runWhenIdle = window.requestIdleCallback || ((cb) => setTimeout(cb, 50));
```
